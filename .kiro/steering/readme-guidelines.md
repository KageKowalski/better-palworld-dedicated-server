# README Guidelines

README.md is a practical document for human users. It should focus on:

- A brief description of what the project does
- Installation instructions
- Usage instructions

Any technical details included must be short and succinct. Avoid lengthy architecture explanations, internal design rationale, or implementation specifics — those belong in steering files or spec documents, not in the README.
